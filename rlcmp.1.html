NAME
rlcmp — recursive file compare

SYNOPSIS
rlcmp [-agmtu] [-d depth] [--] file1 file2

DESCRIPTION
“rlcmp” means “recursive symbolic link compare”. Compared to diff(1) which always follows symbolic links, rlcmp is intended for a
recursive directory compare which never follows symbolic links.

rlcmp stops a file compare when the first difference is found. It acts on files like a very simple version of cmp(1).

OPTIONS
--
Treat following arguments as filenames even if they begin with ‘-’.

-aSame as option -gmtu.

-d depth
Limit recursion depth. A depth of 0 means that directories are only entered when they are given as a command line argument.

-gCompare file group.

-mCompare file permissions (except for symbolic links).

-tCompare modification time (except for directories).

-uCompare file owner.

EXIT STATUS
rlcmp exits 0 if no difference is found, 1 on a difference, and 2 if an error occurs.

BUGS
rlcmp is primarily intended to compare regular files, symbolic links and directories. By intention rlcmp does currently not always
sort it’s output. Sorting will only be implemented if there is good reason.

SEE ALSO
diff(1), cmp(1)
